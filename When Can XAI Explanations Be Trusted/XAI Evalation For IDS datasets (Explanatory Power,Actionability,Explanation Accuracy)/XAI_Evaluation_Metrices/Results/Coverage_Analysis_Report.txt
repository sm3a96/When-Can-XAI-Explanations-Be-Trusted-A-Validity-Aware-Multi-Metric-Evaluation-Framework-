2. ROOT CAUSE ANALYSIS
──────────────────────────────────────────────────────────────────────
  Issue #5 — DL SHAP small sample size:
    Root cause: KernelExplainer requires O(n × d²) evaluations.
    With d=66 (CIC) or d=56 (IDS) features and slow DL inference,
    n was capped at 50 (IDS) or 200 (CIC).
    CIC (n=200): CI crosses zero for all DL SHAP → INCONCLUSIVE_NOISE
    IDS (n=50):  Minimal adequacy → INCONCLUSIVE_NOISE
    Finding: DL SHAP computational cost is itself a paper finding.
    Paper claim: 'KernelExplainer on DL models is computationally
    infeasible for n>50 at inference time, limiting EP reliability.'

  Issue #6 — Anchors incomplete comparison:
    Root cause A (generation gap): Anchors not generated for RF/IDS2025.
      RF with Anchors at n=30 would have been feasible but was not run.
      This is a documentation gap acknowledged in the paper.
    Root cause B (method inapplicability): Anchors is a rule-based method
      requiring a prediction function; DL Anchors not implemented.
    Consequence: Anchors has only 1 RELIABLE row (RF/CIC, n=200, d=0.91).
    Paper claim: 'Anchors coverage is limited to classical models;
      cross-dataset comparison requires future work.'
